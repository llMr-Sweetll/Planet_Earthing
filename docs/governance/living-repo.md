# Living Repo Practice

Planet_Earthing should change as evidence improves. The point of a living repo is not to keep moving the goalposts; it is to keep decisions visible.

## Versioned Artifacts

Freeze versions for:

- preregistration,
- protocol,
- sensor suite,
- data dictionary,
- feature extraction code,
- analysis plan.

Use a new version when a change affects interpretation.

## Decision Records

Major decisions should be added to [decisions.md](decisions.md):

- species selection,
- sensor additions/removals,
- protocol deviations,
- analysis changes,
- claim language changes.

## Claim Labels

Use:

- `supported`
- `plausible-but-unproven`
- `exploratory`
- `unsupported`
- `superseded`

Every public-facing claim needs one of these labels.

## Change Discipline

Before data collection:

- improve freely,
- cite sources,
- record rationale.

After preregistration freeze:

- separate planned analyses from exploratory analyses,
- record deviations,
- never rewrite the original protocol as if it had always said the new thing.
