# Maintainer Guide

Version: 0.1.0

## Maintainer Role

Maintainers protect the project's clarity, evidence standards, contributor experience, and participant privacy.

## Review Priorities

Review pull requests in this order:

1. Privacy and consent risk.
2. Claim accuracy.
3. Experimental control and reproducibility.
4. Code correctness and tests.
5. Clarity for new contributors.

## Merge Criteria

Merge when:

- the change is scoped,
- relevant validation passes,
- sources are linked for scientific claims,
- no sensitive data are included,
- protocol or schema changes are reflected in templates,
- maintainers agree the uncertainty is represented honestly.

## Release Tags

Use tags for important research freezes:

```text
protocol-v0.1.0
pilot-freeze-v0.1.0
analysis-freeze-v0.1.0
```

Every freeze should include:

- README state,
- protocol state,
- data dictionary state,
- analysis configuration,
- decision log.

## Handling Public Attention

If the project receives attention because of sensational claims:

- point to the fact check,
- repeat the exploratory status,
- clarify that plant signaling is real but familiar-human recognition is unproven,
- avoid debating consciousness claims beyond the documented evidence posture.
