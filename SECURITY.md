# Security And Privacy

Planet_Earthing does not currently ship production software, but the project may handle sensitive research data.

## Do Not Report Publicly

Please do not post any of the following in public issues or pull requests:

- participant names,
- consent forms,
- raw video or audio,
- private sensor files,
- private repository tokens,
- exact locations of study rooms or homes,
- unreleased participant-level datasets.

## Sensitive Data Handling

Raw human-subject data should be stored outside git in access-controlled storage. Public releases should use de-identified metadata, derived features, synthetic examples, or consent-reviewed datasets.

## Vulnerability Or Privacy Issue

If you find a privacy leak or security issue, contact the repository owner privately through GitHub if available. If private contact is not available, open a minimal public issue that says a privacy/security concern exists, without including the sensitive details.
